正規語言詳細解析：CH5 - CH7

本文件針對講義每張投影片的觀念進行深入展開解釋，並附上定理證明邏輯與詳細的例題解題過程。

Chapter 5: Context-Free Languages

上下文無關文法 (Context-Free Grammars) 介紹

投影片內容解析：許多實用的語言（如大部分的程式語言）都不是正規語言 (Regular Languages)，我們需要表達能力更強的模型，也就是「上下文無關文法 (CFG)」。

CFG 的核心特徵是：生成規則的左側必須是單一變數 (Single Variable)，而右側沒有任何限制（可以是變數與終止符號的任意組合）。若一個語言能由 CFG 產生，我們便稱之為上下文無關語言 (CFL)。因為正規文法是 CFG 的一種嚴格受限形式（右側只能是單一終止符號或終止符號配變數），所以正規語言是 CFL 的「真子集 (Proper Subset)」。

CFG 經典例題解析

投影片內容解析：這裡提供了三個經典的 CFG 構造例題，展示如何將語言邏輯寫成文法規則。

例題 1：偶數長度的回文 (Even-length palindromes)

語言：L = {wwR : w ∈ {a, b}*}

V = { S }, T = { a, b } S → aSa | bSb | λ

解題過程：回文的特性是從兩端往中間對稱。因此我們同時在字串的左右兩側生成相同的符號 aSa 或 bSb。當到達中心時，用 λ 結束推導。例如 aabbaa：S ⇒ aSa ⇒ aaSaa ⇒ aabSbaa ⇒ aabbaa。

例題 2：a 與 b 數量相等

語言：L = {w ∈ {a, b}* : na(w) = nb(w)}

S → aSb | bSa | SS | λ

解題過程：每生成一個 a 就必須配對一個 b，所以有 aSb 和 bSa。但字串可能是並列的兩個平衡字串（例如 abab），所以需要 SS 規則來串聯兩個平衡字串。

最左推導、最右推導與推導樹 (Theorem 5.1)

投影片內容解析：在產生字串的過程中，如果我們每一步都強制替換「最左邊」的變數，稱為最左推導 (Leftmost Derivation)；反之稱為最右推導 (Rightmost Derivation)。無論是哪種推導順序，最終產生的字串是相同的。

推導樹 (Derivation Tree / Parse Tree)：以圖形方式表示推導過程。根節點 (Root) 標示為 S，內部節點是變數，子節點是規則右側的符號。對這棵樹進行「最左深度優先遍歷 (Leftmost depth-first traversal)」所得到的葉節點字串，稱為該樹的產出 (Yield)。

Theorem 5.1

給定一個 CFG G，對於 L(G) 中的每一個字串 w，必定存在一棵推導樹，其產出為 w；反之，任何一棵以 G 的規則生成的推導樹，其產出必屬於 L(G)。這證明了文法的語法結構與樹狀結構是一一對應的。

剖析與成員問題 (Parsing & Membership)

投影片內容解析：剖析問題是指給定字串 w，找出是否有一連串的推導能產生 w。

窮舉剖析 (Exhaustive/Brute-force Parsing) (Theorem 5.2)：嘗試所有可能的推導。它能保證找出屬於語言的字串，但如果字串「不在」語言中，且文法包含 A → λ 或 A → B，程式可能會因為長度未增加而陷入無窮迴圈。

窮舉法的時間複雜度非常高，最壞情況下會產生 O(|P|2|w|+1) 也就是指數級別的節點。

Theorem 5.3 與線性時間剖析

對於任何 CFG，存在演算法能在 O(|w|³) 時間內完成剖析 (例如 CYK 演算法)。

若文法是 S-grammar (Simple-grammar)，即所有規則都是 A → ax（a 是終止符，x 是變數字串），且每一個 (A, a) 配對最多出現一次，則因為每讀一個字元就能唯一決定下一條規則，剖析時間可降至線性時間 O(|w|)。

文法的模擬兩可性與消除 (Ambiguity)

投影片內容解析：一個文法被稱為模擬兩可 (Ambiguous)，如果語言中存在某個字串 w，它擁有兩棵以上的推導樹（或者說擁有兩個不同的最左推導）。

在程式語言中，模擬兩可是不被允許的。例如運算式 a+b*c 如果用 E → E+E | E*E | a | b | c 解析，可能會先加後乘，也可能先乘後加，導致語意錯誤。

消除模擬兩可的解題過程：

我們透過引入「層次」來強迫運算先後順序（例如：乘法優先於加法）：

E → E+T | T (Expression 層：處理加法) T → T*F | F (Term 層：處理乘法) F → (E) | I (Factor 層：處理括號與變數) I → a | b | c

如此一來，a+b*c 的乘法 b*c 必定會在較底層的 T → T*F 先被綁定，確保了唯一且正確的推導樹。

注意：有些語言是先天模擬兩可 (Inherently Ambiguous) 的，例如 {anbncm} ∪ {anbmcm}，這種語言無論你怎麼改寫文法，都一定會產生模擬兩可的情形。

Chapter 6: Simplification and Normal Forms

文法轉換與替換法則 (Theorem 6.1)

投影片內容解析：為了演算法的高效執行與分析，我們經常需要限制 CFG 右側的形式，這就是「簡化」的目的。簡化不一定會減少規則總數，但能讓文法結構更乾淨。

Theorem 6.1: 替換法則 (Substitution Rule)

若有規則 A → uBv，且 B → y1 | y2 | ... | yn。我們可將 A → uBv 移除，直接替換為 A → uy1v | uy2v | ... | uynv。這確保了生成語言不變，是簡化過程的基礎。

消除無用生成 (Removing Useless Productions)

投影片內容解析：一個變數是「無用的 (Useless)」，如果它滿足以下任一條件：

  1. 無法終止：它無法推導出任何純終止符號的字串。
  2. 不可到達：從起始符號 S 出發，無法推導出包含該變數的句型。
例題：消除無用生成

文法： S → aS | A | C, A → a, B → aa, C → aCb

步驟 1 (找出能終止的變數 V1)：
A → a (A 加入 V1)。B → aa (B 加入 V1)。S → A，因為 A 在 V1，所以 S 加入 V1。C 的規則右側永遠包含 C，無法產生純終止符，所以 C 是無用的。移除包含 C 的規則。
剩下： S → aS | A, A → a, B → aa。

步驟 2 (相依圖與可達性)：
從 S 出發可到達 S 與 A。但 B 完全無法被 S 到達。因此 B 也是無用的。
最終結果： S → aS | A, A → a。

消除 λ-生成 (Removing λ-Productions)

投影片內容解析：形式為 A → λ 的規則稱為 λ-生成。若文法語言本身不含 λ，我們可以消除這些規則。

作法：先找出所有「可空變數 (Nullable variables)」（能推導出 λ 的變數）。然後在其他規則中，將右側出現的可空變數進行「有存在」與「被消去」的所有排列組合替換，最後移除原先的 A → λ。

例題：消除 λ-生成

文法： S → ABaC, A → BC, B → b | λ, C → D | λ, D → d

1. 找出 Nullable：B, C 都有 λ。因為 A → BC，所以 A 也是 Nullable。
2. 移除 λ 規則。
3. 對 S → ABaC 進行組合展開 (A, B, C 皆可空)：
包含全存在：ABaC
消去一個：BaC (消A), AaC (消B), ABa (消C)
消去兩個：aC (消AB), Ba (消AC), Aa (消BC)
消去三個：a (消ABC)
結果： S → ABaC | BaC | AaC | ABa | aC | Ba | Aa | a。

消除單一生成與簡化順序 (Theorem 6.4 & 6.5)

投影片內容解析：形如 A → B 的規則稱為單一生成 (Unit-productions)。這會增加剖析的無謂步驟。

消除作法：畫出相依圖，找出所有 A 到 B 的路徑。如果 A 能到達 B，就把 B 所有非單一生成的規則，直接複製給 A。

Theorem 6.5: 正確的簡化順序

非常重要！消除一種規則可能會產生另一種規則，因此必須嚴格遵守以下順序：

  1. 消除 λ-生成 (因為消除它可能會產生新的單一生成或無用變數)
  2. 消除單一生成
  3. 消除無用生成

Chomsky 正規形式 (Chomsky Normal Form, CNF)

投影片內容解析：在 CNF 中，所有規則的右側被嚴格限制為只有兩種形式：

  • A → BC (兩個變數)
  • A → a (單一終止符號)
演算法與例題：轉換為 CNF

文法： S → ABa, A → aab, B → Ac

Step 1: 將終止符轉為變數
新增 X → a, Y → b, Z → c。
替換原規則：S → ABX, A → XXY, B → AZ。

Step 2: 將長度超過 2 的變數串列截斷
S → ABX 拆為：S → AC, C → BX。
A → XXY 拆為：A → XD, D → XY。

最終 CNF： S → AC, C → BX, A → XD, D → XY, B → AZ, X → a, Y → b, Z → c。

Greibach 正規形式與左遞迴消除 (GNF)

投影片內容解析：在 Greibach Normal Form (GNF) 中，規則右側必須是一個終止符號開頭，後面跟著任意數量的變數 (A → aV*)。這種形式非常適合由上而下剖析，因為每一步都能消耗一個輸入字元。

消除左遞迴 (Left Recursion to Right Recursion)：如果文法有 A → Ax | y，會導致由上而下剖析陷入無窮迴圈。必須轉換為右遞迴：
A → yA'
A' → xA' | λ

Chapter 7: Pushdown Automata (PDA)

非決定性下推自動機 (Nondeterministic Pushdown Automata)

投影片內容解析：有限狀態機 (FSA) 沒有記憶體，無法處理如 anbn 這種需要計數的語言。下推自動機 (PDA) 在 FSA 的基礎上，加入了一個堆疊 (Stack) 作為儲存機制，使其具備處理上下文無關語言 (CFL) 的能力。

NPDA 包含七個元素：狀態集合 (Q)、輸入字母表 (Σ)、堆疊字母表 (Γ)、轉換函數 (δ)、初始狀態 (q0)、堆疊底部符號 (z) 以及最終狀態集合 (F)。

NPDA 的轉換、圖形與接受模式

投影片內容解析：NPDA 的轉換函數 δ(q, a, b) = {(q', cd)} 代表：當前在狀態 q，讀入符號 a，且堆疊頂端為 b 時，轉移到狀態 q'，並將堆疊頂端的 b 彈出，壓入 cd。

接受模式 (Modes of Acceptance)：字串被接受有三種等價的定義方式：

  1. 到達最終狀態 (Accepted on final state)：讀完所有輸入後，狀態位於 F 中。
  2. 清空堆疊 (Accepted on empty stack)：讀完所有輸入後，堆疊完全清空（包含 z 都不見）。
  3. 兩者皆是。

我們通常使用「瞬時描述 (Instantaneous Descriptions, ID)」 (q, w, u) 來追蹤 PDA 的執行過程（當前狀態、剩餘字串、堆疊內容）。

CFG 與 NPDA 的等價性 (Theorem 7.1)

投影片內容解析：Theorem 7.1 證明了上下文無關語言 (CFL) 與 NPDA 的等價性：對於任何一個 CFL，必定存在一個 NPDA 可以辨識它。其構造性證明是利用 Greibach Normal Form (GNF)。

從 GNF 構造 NPDA

投影片內容解析：因為 GNF 的特性是每次推導都會產生一個終止符號（對應消耗一個輸入字元），所以可以非常直覺地轉為 NPDA：變數存放在堆疊中，輸入字元用來觸發轉換。

例題：GNF 轉 NPDA

GNF 文法： S → aSA | a, A → bB, B → b

構造規則：

1. 初始啟動：δ(q0, λ, z) = {(q1, Sz)} // 將 S 壓入堆疊 2. 模擬文法 (在 q1 狀態)： 對應 S → aSA：δ(q1, a, S) 加入 (q1, SA) 對應 S → a： δ(q1, a, S) 加入 (q1, λ) 對應 A → bB： δ(q1, b, A) = {(q1, B)} 對應 B → b： δ(q1, b, B) = {(q1, λ)} 3. 接受結束：δ(q1, λ, z) = {(qf, λ)} // 堆疊空到剩 z，進入最終狀態

如果是「任意 CFG」轉 NPDA，則先不消耗輸入就把變數展開到堆疊，遇到終止符號時，再用 δ(q, a, a) = {(q, λ)} 彈出匹配的終止符號。

將 NPDA 轉換為 CFG

投影片內容解析：將 NPDA 轉回 CFG 的過程非常繁雜。概念是創造形如 (qi A qj) 的複合變數，代表「從狀態 qi 開始，清空堆疊符號 A，最後到達狀態 qj」的過程。

例如，若有轉換 δ(qi, a, A) = (qj, BC)，則產生的文法規則會是：
(qi A qk) → a (qj B ql) (ql C qk) （對所有可能的狀態 qk, ql 展開組合）。接著再利用 CH6 學到的「消除無用生成」把不可能到達的狀態路徑刪除。

決定性下推自動機 (DPDA) 與 DCFL

投影片內容解析：DPDA 是 NPDA 的特例，要求在任何組態下，最多只能有一種合法的轉換選擇（不能有分岔，也不能在有 λ-move 時存在輸入字元的衝突）。

決定性上下文無關語言 (DCFL)：能被 DPDA 接受的語言。請注意，DPDA 的能力嚴格小於 NPDA！例如回文 wwR 是 NPDA 可以辨識的，但 DPDA 無法辨識（因為 DPDA 無法猜測字串的中間點在哪裡）。

DCFL 的重要性在於，它們可以使用線性時間的高效演算法進行剖析 (如 LR parser)，這正是現代編譯器 (Compiler) 解析程式碼的基礎理論。
