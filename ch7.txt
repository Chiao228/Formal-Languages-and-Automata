The Language Accepted by a Pushdown Automaton The language accepted by a npda is the set of all strings that cause the npda to halt in a final state, after examining all input symbols The final contents of the stack are irrelevant As was the case with nondeterministic automata, the string is accepted if any of the computations cause the npda to halt in a final state

Pushdown Automata and Context-Free  Languages Theorem 7.1 states that, for any context-free language L, there is a npda to recognize L Assuming that the language is generated by a context-free grammar in  Greibach normal form , the constructive proof provides an algorithm that can be used to build the corresponding npda The resulting npda simulates grammar derivations by keeping variables on the stack while making sure that the input symbol matches the terminal on the right side of the production
